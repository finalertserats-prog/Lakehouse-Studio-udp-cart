LakeHouse Studio Accessibility Conformance Report

Based on VPAT® Version 2.5 — WCAG Edition
Report date: July 07, 2026  |  Product: LakeHouse Studio v0.6 (web UI, frontend/index.html)  |  Prepared by: Techsophy Data Engineering

Product Description

LakeHouse Studio is a browser-based installer and operations console for open data lakehouse stacks. The evaluated surface is the single-page web application served at the product root (dark-theme Tailwind UI: 8-step install wizard, Stack Builder, live install log streaming, and day-2 operations panels). The lks CLI is a text-based alternative interface and is not covered by this report.

Evaluation Methods Used

  • Full static source audit of frontend/index.html (~6,200 lines): semantic structure, ARIA usage, form labelling, keyboard handlers, focus styling, animation and contrast token analysis.
  • Colour-contrast calculation of the theme's Tailwind text/background token pairs against WCAG ratios.
  • Functional walkthrough of the running product (install wizard end-to-end) on Chromium.
  • Limitation: no assistive-technology testing (NVDA / JAWS / VoiceOver) was performed. Results for screen-reader criteria are derived from code analysis and should be confirmed with AT testing before publishing this VPAT externally.

Conformance statement: LakeHouse Studio v0.6 does not currently conform to WCAG 2.1 Level A or AA. The HTML foundation is strong (native buttons and form controls, alt text, unblocked zoom, logical heading structure, Escape-dismissable dialogs), but the product was built without an assistive-technology layer: there are no live-region announcements, no dialog semantics, no visible focus indication, and several core wizard steps are mouse-only. The five highest-impact gaps are listed under Remediation Priorities.

Remediation Priorities

#FixCriteria unblocked
1Add a visible focus style (e.g. 2px cyan outline on :focus-visible) and remove the three outline:none rules.2.4.7
2Make the card pickers keyboard-operable: render template/stack/tier/goal cards as <button> (or add tabindex=0 + Enter/Space handlers).2.1.1, 4.1.2
3Add aria-live regions: polite for toasts and install state changes, off-screen status for step transitions; give modals role="dialog", aria-modal, focus-on-open and focus-restore.4.1.3, 4.1.2, 2.4.3, 3.3.1
4Associate every label with its control (for=/id) and add autocomplete on user-data inputs.1.3.1, 3.3.2, 1.3.5
5Honour prefers-reduced-motion (disable aurora/confetti animations) and lift text-slate-500 to slate-400 for 12px text; raise hairline border alpha to reach 3:1.2.2.2, 1.4.3, 1.4.11
