How it works
From cart to query in 8 steps

Studio compresses what's normally a 6-week integration into a guided 30-minute flow. Each step is reversible, every error is explained.

1
Choose your goal
Pick the use case closest to yours (local demo / production / AI-ML / analytics / air-gapped). We pre-select sensible defaults but you control every choice.
2
Build your stack
Shop components category by category — table format, catalog, object storage, processing, serving. Live compatibility score (0-100) updates as you add. Alternates listed as "coming soon".
3
Name your lakehouse
Personal name — "ocean", "nebula", "halcyon" — shown everywhere afterward. Suggestions on tap, all validated.
4
Size your server
Three tiers (minimal / recommended / comfortable) sized from your cart. Live-matched VPS plans from Hetzner, DigitalOcean, Linode, Vultr — cheapest fit highlighted, direct signup links.
5
Connect server
Host + install directory. v0.3 runs against the same machine Studio is on; SSH-to-remote-VPS lands in v0.4.
6
Pre-flight inspection
Docker, Compose, bash, RAM, CPU, disk, port conflicts — 14 checks before installing. Never installs into a broken environment.
7
Install with live logs
Clone repo, write env file, run doctor / start / bootstrap / smoke test. Live WebSocket logs in a terminal panel. Per-step Retry, Skip, Rollback if anything fails — with a diagnose panel that explains what / why / how to fix.
8
Run a query
Demo data loaded. One-click queries against StarRocks. Sandboxed SQL editor for ad-hoc reads. Service URLs for MinIO, Iceberg REST, Spark, StarRocks. Confetti.
Authentication
Studio requires an auth token
Paste the value of LHS_AUTH_TOKEN from the server console.
The token is stored in your browser only. Note: in v0.3 the token sits in localStorage — see README for the cookie-auth migration plan.
